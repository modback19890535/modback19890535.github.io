矩陣動作 MATRIX ACTIONS

 

正如其名稱一般無可置疑的是，矩陣動作只在矩陣網路中生效。但這並不是說在矩陣網路中你只能進行這幾個有限的行動。矩陣動作的特殊性在於，它們應用特殊的規則。比如雜訊和監視指數（OS；Overwatch Score）。

 

這一節列舉了所有的矩陣動作。這包括了一大堆用來處理你在數字世界裡可以做的事的規則。不過當然，它們並不是你在矩陣中能進行的唯一行動。如果你想進行一些這些規則未涵蓋的行動，比如把自己的圖示改變得更時髦，或者用數據流創造出令人分心的VR影像等等，那麼你的GM會決定你需要嘗試進行何種檢定。

 

當一臺裝置需要進行防禦檢定，而這個檢定需要一項精神屬性的時候，使用其所有者的對應屬性等級代替。即使此時她本人並沒有參與防禦，甚至完全沒有在操作這臺裝置，她在這之前對裝置的操作和設定已經影響了防禦檢定。如果一臺裝置完全處於無主狀態，它的裝置等級（DR, Device Rating）代替它在防禦檢定中需要且不具有的精神屬性。舉個例子的話，一臺被主人設定好，然後忘到九霄雲外的裝置——比如門鎖——在對抗〈控制裝置 Control Device〉行動的防禦檢定骰池中使用它的DR替代直覺屬性。

 

非法行為 ILLEGAL ACTIONS

有些矩陣動作是非法的，這讓它們比合法行為更具風險。矩陣在被建立時，安全性是考慮因素中的重中之重，但設計者當然知道不可能讓矩陣完全免遭黑客毒手。非法行為的列表非常簡單：所有攻擊性和隱匿性的動作都是非法的。非法行為的風險取決於你執行了哪一類動作。（編注：動作對應檢定的檢定上限來判斷它屬於哪類動作。比如說〈暴力破解〉是攻擊性動作，〈後臺入侵〉是隱匿性動作）

 

攻擊性動作

如果你在攻擊性動作中失敗，你的目標的安全軟體會拒絕你的程式碼，破壞它並把它發送回去。如果只是普通數據，你的系統會檢查錯誤，但在這種情況下，這些被設計用來繞過防火墻的程式碼極其兇猛。你的目標在防禦檢定中得到的每一個凈成功，都會讓你承受1格不可抵抗的矩陣傷害。不過，因為你沒根本能影響到你的對手，所以你不會被注意到。

如果你成功地執行了一個攻擊類動作，你的目標會意識到它正在被另一個圖示攻擊。但它並不能自動地辨識到你。更大的可能性是，它會用自己的下一個動作主動搜索你的存在。然而，它總是會將此次攻擊警告給它的所有者，並且（如果是主機的話）執行IC，取決於其所有者的偏好和GM的裁斷。

 

隱匿性動作

如果你在隱匿性動作中失敗，目標的防火墻軟體會偵測到入侵併在你身上放置一個MARK。裝置會立刻通知它的擁有者，主機會放出IC。如果目標在你身上已經有了3個MARK，它不會再放置更多的MARK，但通知和放出IC仍然照常。

如果你成功地執行了一個隱匿類動作，你的存在不會暴露。

 

非駭客能做的矩陣動作 MATRIX ACTIONS FOR THE NON-HACKER

你不是隊伍里的駭客，但你的角色表有通訊鏈。你可能要問：它有什麼用？用處自然很多，但你在這章中需要記住的不多。這裡有一個你能用通訊鏈做的矩陣動作簡表。

 

你的網域 YOUR GRID

你得知道你在用什麼網域，但如果你沒有特地升級，你的網域基於你的生活方式：

低端或更低：公共網域；

中端：本地網域；

高階：全球網域——在十大巨頭裡面選一個作為你的網域提供者；

奢華：你隨便選吧，老兄。

 

你通訊鏈的矩陣屬性 YOUR COMMLINK』S MATRIX ATTRIBUTES

你的GM偶爾會問你兩項矩陣屬性：數據處理（如果你隊里的駭客沒完成她的活）和防火墻。除非你的通訊鏈是定製的，不然這兩項屬性都等於通訊鏈的裝置等級。記得在你角色單的街頭裝備部分記一下。

 

五項矩陣動作 FIVE MATRIX ACTIONS

作為一個普通的矩陣使用者，你只有五項可能需要看看的矩陣動作。

改變圖示（Change Icon）：如果你喜歡在VR模式下折騰（tooling around），你需要去看看這個動作，得到你想要的外貌。

檔案編輯（Edit File）：如果你要碼字或是剪輯視訊，或者是寫報告，你可能想去了解一下〈編輯檔案〉動作。你還可以用這個動作保護你通訊鏈上的檔案，老練的狂奔者都知道這是個好主意。

矩陣全防禦（Full Matrix Defense）：如果你的通訊鏈或任何與之相連的裝置受到攻擊，你不知道該怎麼做，就用這個動作開啟通訊鏈的所有防禦手段。

矩陣搜索（Matrix Search）：通訊之外，搜索是口袋裡有矩陣的最大優勢。

發送資訊（Send Message）：這個動作可能是你通訊鏈最主要的用途。它並不複雜，但你可以去看一眼。

暴力破解 BRUTE FORCE 

（複雜動作）

許可權需求：無

檢定：矩陣戰鬥＋邏輯【攻擊性】v. 意志＋防火墻

你可以用這個矩陣動作去給你本不具有許可權的一個目標打上MARK。較之〈後臺入侵Hack on the fly〉，採用〈暴力破解〉的黑客們更強調其行為的攻擊性，而不是隱匿性。

如果你在對抗檢定中勝出，你在目標上放置１個MARK，對同一個目標最大能擁有３個MARK。如果你希望的話，你在對抗檢定中獲得的每2個凈成功可以讓你對該裝置造成1點矩陣傷害。這個傷害值的抵抗檢定為，該裝置的DR＋防火墻。

在擲骰前，你可以聲明嘗試同時放置超過一個MARK。如果你試圖在一個動作中放置2個MARK，你會受到-4骰池罰值；如果試圖一次放置3個MARK，那麼骰池罰值變為-10。

你也可以用這個矩陣動作去翻到本不具有合法許可權的網域裡面去。在這種情況下，本地網域的防禦骰池是4個骰子，全球網域的防禦骰池是6個骰子。如果你勝出，你不會在網域上放置MARK，而是立刻翻到目標網域中。成功使用〈暴力破解〉以翻越網域不會像大部分成功的攻擊動作一樣觸發網域的警報，也不會驚動次級GOD。

 

改變圖示 CHANGE ICON

（簡單動作）

許可權需求：所有者（Owner）

檢定：無（數據處理類動作）

你改變目標的圖示，使其成為一個你擁有的樣本的複製品，或一個你自己設計的圖案。改變圖示並不會改變〈矩陣偵察Matrix Perception〉動作的結果，但可能會矇蔽那些沒有花時間觀察你的新外表的化身們。你可以把自己的圖示作為〈改變圖示〉行動的目標。

 

查詢監視指數 CHECK OVERWATCH SCORE

（簡單動作）

許可權需求：無

檢定：電子戰＋邏輯【隱匿性】v. 6

你可以查明網域離開始對你發動合圍（Converging）還有多遠。檢視監視指數（OS）是一個隱匿動作，因此對抗檢定中對方獲得的成功數會加入到你的OS中。如果你在檢定中勝出，GM會告知你開始這個動作時的OS，然後再把防禦骰池中的成功數加到你的OS上。

 

控制裝置 CONTROL DEVICE 

（複雜動作）

許可權需求：1(自由動作)／2(簡單動作)／3(複雜動作)

檢定：（視情況）【數據處理(/特殊，見詳述)】v. （視情況），或

電子戰＋直覺【隱匿性】v. 直覺＋防火墻

通過像使用遙控器或遊戲機手柄一樣操作你的數據板或通訊鏈，你可以操縱你控制（或有足夠控制權）的一臺裝置完成一個動作。你嘗試進行的行動所需的檢定骰池使用你在正常情況下進行此項檢定所需的技能等級和屬性等級。舉例來說，

操縱無人機搭載的武器朝目標開火要進行檢定，炮術＋敏捷；

操縱一臺遙控式水下焊接機器人需要進行檢定，航海機械＋邏輯。

控制裝置期間進行的檢定應用該檢定的正常檢定上限，或你的數據處理等級，取其較低者。如果沒有適合你想讓裝置進行的檢定型別，比如解鎖一個磁力鎖，或者彈出手槍的彈匣，你必須進行贏得對抗檢定，電子戰＋直覺【隱匿性】v. 直覺＋防火墻，而不必繼續做出其他檢定。

你可以使用這個矩陣動作同時操控複數臺裝置。如果每一臺裝置都為你所擁有，且它們都被命令在同一時間進行相同的行動，那麼這不需要額外花費一些什麼；否則，你必須按照你想在一個動作中操縱的裝置個數來分裂骰池。

這個動作的型別（比如，自由動作/簡單動作/標準動作/複雜動作）與受控裝置嘗試進行的動作相同，其所需要的許可權為：自由動作1個MARK，簡單動作2個MARK，複雜動作3個MARK。

當你在〈控制裝置〉的檢定中使用隱匿性來作為檢定上限時，〈控制裝置〉成為隱匿動作，這會帶來與一切隱匿動作相同的風險——OS指數的上升，以及失敗所造成的結果。（譯註：隱匿動作失敗會讓你的目標獲得一個對你的MARK，裝置會通知其所有者，主機會執行IC反侵略。）

 

檔案解密 CRACK FILE

（複雜動作）

許可權需求：1個MARK

檢定：駭入＋邏輯【攻擊性】v. 保護等級×2

你去除掉目標檔案上附加的保護措施，使其成為可讀取的檔案。當然，不需要對一個沒有保護的檔案執行這項動作。

 

程式破壞 CRASH PROGRAM

（複雜動作）

許可權需求：1個MARK

檢定：矩陣戰鬥＋邏輯【攻擊性】v. 直覺＋防火墻

你將目標的儲存空間部分過載，以此來干擾它正在執行的程式。你需要明確指出哪一個程式是你想要破壞的。你可以通過〈矩陣偵察〉動作，或是通過觀察正在生效的程式效果來確定你的目標究竟在執行哪些程式。如果你在對抗檢定中勝出，該程式被終止，並且不能在該裝置上再次執行，直到該裝置進行一次重啟。

 

數據棘刺 DATA SPIKE

（複雜動作）

許可權需求：無

檢定：矩陣戰鬥＋邏輯【攻擊性】v. 直覺＋防火墻

你向一個化身或是一臺裝置發射一段有害的指令，由此對目標造成矩陣傷害。該次攻擊的基礎DV為｛攻擊性｝；每個凈成功使DV＋１，以及每個你對目標擁有的MARK使DV＋２。這項傷害是矩陣傷害（見228頁），抵抗檢定為，目標的DR＋防火墻。

 

解除數據炸彈 DISARM DATA BOMB

（複雜動作）

許可權需求：無

檢定：軟體＋直覺【防火墻】v. 數據炸彈等級×2

你嘗試解除一個你已經偵測到（通常作為〈矩陣偵察〉檢定的結果）的數據炸彈。如果你得到了至少１個凈成功，那麼數據炸彈被安全地解除並刪除。否則，數據炸彈會被觸發，造成它所設定好的傷害，還有可能會摧毀它所保護的檔案（如果這顆炸彈是設計用來摧毀檔案的話）。

 

檔案編輯 EDIT FILE 

（複雜動作）

許可權需求：1個MARK

檢定：計算機＋邏輯【數據處理】v. 直覺＋防火墻

〈檔案編輯〉允許你創造、篡改、複製、刪除或保護任意種類的檔案。這個檢定的防禦者是持有這個檔案的主機或檔案的所有者。每個〈檔案編輯〉動作可以改變目標檔案的一處細節——一條文字的一小段，一幅影象的一個角落，或者是視訊音訊中的兩三秒。你可以與GM一起協商「一處細節」究竟包括哪種程度。GM可能會因為你做出的編輯過於複雜或棘手而在你的檢定上應用罰值。如果你希望進行連續性的編輯，比如從一段視訊上抹掉隊友們的身影，那麼你需要在每個戰鬥輪中進行一次〈檔案編輯〉，直到你停止繼續編輯為止。

如果你用這個動作複製一份檔案，那麼你會成為新複製的檔案的所有者。如果你嘗試複製的檔案具有保護措施，那麼〈檔案編輯〉自動失敗。如果這份檔案設有數據炸彈，那麼炸彈會在你身上炸開花（所以務必記住在打開檔案之前掃瞄它們）。

如果你是檔案的所有者，你也可以用〈檔案編輯〉來為你的檔案設定保護。如果你希望保護一份檔案，做一個計算機＋邏輯【數據處理】的簡單檢定，你得到的成功數成為受保護檔案的等級。一個受保護的檔案不能被讀取、更改、刪除或複製，除非保護被破壞。（譯註：去除檔案的保護，需要使用〈檔案解密〉矩陣動作。得知其是否被保護需要進行〈矩陣偵察〉動作。）

 

進入/退出主機 ENTER/EXIT HOST

（複雜動作）

許可權需求：1個MARK

檢定：無

你進入你擁有MARK的一臺主機，你的圖示會出現在那裡面；你也可以用這個動作從一個主機中離開。這並不需要檢定：主機允許得到MARK的人任意進入，已在內部的人也可以隨意退出。但當你駭入主機時，它可能不會這麼熱情好客。當然，有些IC（反入侵元件）可以把你困在主機里。那時除非強行闖出，否則你無法使用此動作離開主機。

當你離開主機時，你回到進入主機之前你所在的網域。

 

抹除MARK ERASE MARK

（複雜動作）

許可權需求：特殊

檢定：計算機＋邏輯【攻擊性】v. 意志＋防火墻

你除去一個已經放置在你自己的化身或其他圖示上的一個MARK。爲了如此做，你必須擁有想抹除MARK的目標的3個MARK。然而，你不需要擁有該MARK放置者的MARK。你可以以-4骰池罰值同時試圖抹去2個MARK，或以-10骰池罰值嘗試同時抹去3個。如果你試圖一次性抹去多個MARK，那麼這些MARK必須是由同一個圖示所產生、並放置在同一個目標圖示之上的。你不能用這個動作來改變裝置的所有者。

舉個例子，一個IC程式已經MARK了你和你的同伴，你進行檢定，你的【攻擊性】v. IC的等級＋防火墻，來抹除它放置在你身上的MARK。但你需要對你同伴的圖示擁有3個MARK許可權，才能抹除他身上的IC的MARK。可惜他不是喜歡分享的那種人，所以現在他活該了。（譯註：即，對碟客PC而言，組隊主動對你的同伴打MARK，是你RP的一部分——怎麼做到，能否做到就是你的事了；對GM而言，GM有需要引導碟客PC對他的同伴打MARK，或者抓住這個機會教育他們）

 

抹除矩陣簽名 ERASE MATRIX SIGNATURE

（複雜動作）

許可權需求：無

檢定：計算機＋共鳴【攻擊性】v. 簽名等級×２

你除去一個共鳴體（Resonance Being）——如超鏈者或網精——在矩陣中留下的簽名印記。如果你在檢定中勝出，那個簽名就會消散。你必須具有共鳴等級才能嘗試這項動作。但與其他需要共鳴能力的動作不同，這是一個真正的矩陣動作，還是一種攻擊類動作。所以使用它時你有承受矩陣傷害的風險，還會提高監視指數。

 

格式化裝置 FORMAT DEVICE

（複雜動作）

許可權需求：3個MARK

檢定：計算機＋邏輯【隱匿性】v. 意志＋防火墻

你重寫了目標裝置的啟動程式碼。下次這臺裝置重啟時，它反而會永久關機，直到它的軟件系統被更換。更換軟件系統是一個長期檢定，軟體＋邏輯【精神】(12，1小時)。

以此方法關機的裝置失去它的一切無線功能，也不能從矩陣接入或訪問，但依然具有正常的機械功能（比如說，一扇有手動把手的門依然可以被打開，一把有扳機的槍依然可以開火等）。

 

矩陣全防禦 FULL MATRIX DEFENSE

（打斷動作）

許可權需求：無

檢定：無（防火墻類動作）

這個動作允許你對攻擊動作（Attack action）做出防禦，可以在任何時候執行。每當你對抗一個矩陣動作（Matrix Action）而進行防禦檢定時，你可以把你的意志加到骰池中去（如果已經有了，就再加一遍）。

每當你採用〈矩陣全防禦〉動作，你的主動性值-10，它的效果持續到目前戰鬥輪結束。

 

翻越網域 GRID HOP

（複雜動作）

許可權需求：無

檢定：無（數據處理類動作）

你翻越到另一個網域。例如，你可以從公共網域翻到西雅圖翡翠城的地方網域。要想這樣做，你必須擁有目的地網域的接入許可權。如果你沒有許可權，你可以用〈暴力破解〉或者〈後臺入侵〉來非法翻越至其他網域。

如果你身處主機之中，在翻越到其他網域之前你必須離開該主機。

 

後臺入侵 HACK ON THE FLY

（複雜動作）

許可權需求：無

檢定：駭入＋邏輯【隱匿性】v. 直覺＋防火墻

你可以跳過正常合法的步驟，用這個動作來強行在一個目標上放置MARK。與〈暴力破解〉相比，使用〈後臺入侵〉的黑客們更注重她們的隱匿性，而不是破壞力。

當〈後臺入侵〉的目標為一個圖示時，你可以在它上面放置一個MARK，一個圖示上最多有3個MARK。另外，你在對抗中獲得的每2個凈成功數可以視為在〈矩陣偵察〉檢定中獲得的1個成功數，以此來獲得目標的一些資訊。

在擲骰前，你可以聲明嘗試同時放置超過一個MARK。如果你試圖在一個動作中放置2個MARK，你會受到-4骰池罰值；如果試圖一次放置3個MARK，那麼骰池罰值變為-10。

你也可以用這個矩陣動作去翻到本不具有合法許可權的網域裡面去。在這種情況下，本地網域的防禦骰池是4個骰子，全球網域的防禦骰池是6個骰子。如果你勝出，你不會在網域上放置MARK，而是立刻翻到目標網域中。依此法使用〈後臺入侵〉並在檢定中失敗並不會如同其他失敗的隱匿類動作一樣驚動網域或次級GOD。

 

隱藏 HIDE

（複雜動作）

許可權需求：0

檢定：電子戰＋直覺【隱匿性】v. 直覺＋數據處理

即使正在靜默執行（Running silent），你也可能會被其他的圖示辨識（spotted）。如果你在該動作的對抗中勝出，一個目標將失去對你的辨識。如果它希望繼續尋找你，則需要通過進行一個新的〈矩陣偵察〉對抗檢定。你不能對一個在你身上擁有MARK的圖示使用〈隱藏〉動作，所以在使用這個動作之前別忘了清除掉那些MARK。

（簡單動作）

檢定：無（數據處理類動作）

如果你擁有一臺裝置、一份文件、一個化身、一臺裝置主機或一個IC程式，你可以邀請其他的圖示們在其上放置MARK。當你發出邀請時，你選擇可以允許的MARK數量、持續時間以及此次邀請的有效期。受邀者可以用一個自由動作在你允許的圖示上放置MARK。

你可以在對方放置MARK之前的任何時間撤銷邀請，但對方一旦放置了MARK，在你設定好的有效期截止之前，你只能用〈抹除MARK〉，或者通過重啟裝置來清除掉MARK。

 

彈出 JACK OUT 

（簡單動作）

許可權需求：無

檢定：硬體＋意志【防火墻】v. 邏輯＋攻擊性

這個動作將你彈出矩陣，並重啟你正在使用的裝置。如果你正在使用VR，那麼你會承受［甩致休克］。僅當你被誰鏈鎖（link-lock）的時候才會應用防禦骰池。如果多於1個化身導致你被鏈接鎖定，那麼你需要分別擊敗他們每一個人：進行一次檢定，並用你的成功數對比每個對手的成功數。

你只能讓你自己彈出。唯一讓其他人強制跳出的方式是用矩陣傷害把他們打趴下。（編注：有個叫做［數據閥］的小玩意，比這個動作好用到不知道哪裡去）

 

干擾訊號 JAM SIGNALS

檢定：電子戰＋邏輯【攻擊性】

這個動作把你使用的一臺無線裝置變成一個干擾器。如果你不用這臺裝置進行任何其他的矩陣動作，它就會使100米半徑範圍內的雜訊等級增加。你在檢定中獲得的每個成功，都會使雜訊等級＋１。雜訊會影響一切以範圍內裝置為目標的矩陣動作，以及一切經過範圍內裝置傳輸的矩陣動作。如果你想選擇性地干擾某些裝置，或定向干擾，請去買個干擾器——那是它們該乾的活。

矩陣偵察 MATRIX PERCEPTION 

（複雜動作）

許可權需求：無

檢定：計算機＋直覺【數據處理】 （對抗 邏輯＋隱匿性）

這個用途廣泛且重要的動作可以用於兩個方面：尋找矩陣中的圖示，或是分析矩陣中的對象。當你使用〈矩陣偵察〉分析矩陣中的對象或是掃瞄附近的靜默執行中的圖示時，你需要做一個簡單檢定來決定你能獲取多少情報。每個凈成功數可以讓你提出一個關於目標的問題——這可能是目標的型別、等級、MARK數量、其儲存的檔案、其接入的網域、區域內是否有靜默執行的圖示，或者其他任何相關的矩陣資訊。每個凈成功數會讓你知道一項事實。

如果你依此法得知了某裝置上MARK的列表，那麼你只能辨認出你曾經見過的MARK，或被你放置MARK的化身所放置的MARK。此外，你只能得知MARK的數量。

如果你試圖辨識一個距離你100米以上的圖示，這是一個簡單檢定：你獲得的第一個凈成功數允許你發現目標，之後的每個額外的成功數可以用來獲知上述資訊。

如果你在調查某一個靜默執行的圖示（在你確認它存在之後），則需要進行對抗檢定，防禦者用邏輯＋隱匿性 進行檢定。凈成功的使用方式與你辨識遠距目標一樣，即第一個凈成功用來辨識目標，剩餘的用作分析。（譯註：關於矩陣偵察的更多資訊，見〖矩陣偵察〗章節）

裝置重啟 REBOOT DEVICE

（複雜動作）

許可權需求：3個MARK

檢定：計算機＋邏輯【數據處理】v. 意志＋防火墻

被執行此動作的裝置會立刻關機並重新啟動。在下一個戰鬥輪結束時該裝置會重新上線。在它重啟完成之前，此裝置失去一切電子功能，並從矩陣網路中消失。

當你重啟自己化身對應的裝置時，你的OS重置為0，你的一切MARK（無論是你放置在他處的，還是其他人放置在你圖示上的）全部被抹去。如果你在自己處於VR狀態時重啟，你會承受甩致休克（見229頁）。當你返回線上時，你的圖示可能位於任何一個你擁有合法接入許可權的網域中，或者，如果沒有任何網域許可權的話，處於公共網域之中。

當你執行這一行動時，你可以選擇讓目標裝置關機后延遲任意長短的時間，之後再執行啟動工作。任何一個與該裝置有物理接觸的人都可以強行結束這種延遲，只要他們按一下電源鍵就可以讓裝置開始執行，並在下一個戰鬥輪結束時讓該裝置連線上線。

這個動作只對裝置和你自己的化身生效，但對主機、生命體（如超鏈者，儘管他們可以「重啟」自己，見251頁）、共鳴組構體（如網精）無效。

如果你是裝置的擁有者，你不需要進行檢定。

你不能對被［鏈接鎖定］的裝置使用〈裝置重啟〉。

 

發送資訊 SEND MESSAGE

（簡單動作）

許可權需求：不適用（N/A），或1個MARK

檢定：無（數據處理類動作）

你通過矩陣網路向一個你擁有其通訊碼的使用者發送一段資訊。該資訊可以是文字或音訊組成的一個短句，也可以是一幅影象或是一個檔案。如果你通過直接神經界面（DNI）接入矩陣——即使你只是在使用AR模式——那麼就可以傳送更長或更復雜的資訊，比如一長段文字。你也可以使用這個動作來為一個或多個參與者直播（Live feed），你們可以通過任何一種數字錄製裝置來達成這個目的。（譯註：即，通過矩陣共享攝像頭裡的畫面或者數字錄音機傳來的音像等進行直播，或者其他的發送/接收裝置傳遞氣味、觸感等。碟客可以利用這個來建個聊天室/討論組/群。）

 

設定數據炸彈 SET DATA BOMB

（複雜動作）

許可權需求：1個MARK

檢定：軟體＋邏輯【隱匿性】v.（裝置等級×2）

你在一份檔案中設定一個數據炸彈。當你進行這個動作時，你需要為數據炸彈設定等級，最高等級為你在檢定中獲得的凈成功。你還需要選擇炸彈引爆時是否會刪除掉它所附屬的檔案。最後，為你的數據炸彈設定一個密碼，輸入密碼可以避免炸彈的觸發。一份檔案同時只能設定一個數據炸彈。

當某人試圖在未輸入已設定好的密碼的情況下閱讀、編輯、拷貝、保護、刪除該檔案，或在該檔案中設定另一個數據炸彈時，炸彈就被觸發。

數據炸彈的引爆會對落入陷阱的圖示造成｛數據炸彈的等級個D6｝點矩陣傷害（如常抵抗）。按照設定的不同，爆炸的炸彈還可能會刪除該檔案，然後刪除它自己。如果在進行上述會觸發炸彈的行動之前輸入了正確的密碼，那麼數據炸彈不會啟動，但依然附著在該檔案之上，等待著下一個使用者。（譯註：大寫D6的時候，得出來的值要全部加在一起計算總值，而不是計算成功值。矩陣火球術）

數據炸彈可以通過〈矩陣偵察〉動作被偵測到。如果它被偵測到了，其他人就可以用〈解除數據炸彈〉動作來使其無力化。一個被解除的數據炸彈會被刪除掉，該死的。

 

窺探 SNOOP

（複雜動作）

許可權需求：1個MARK

檢定：電子戰＋直覺【隱匿性】v. 邏輯＋防火墻

這個動作使你可以攔截目標裝置發送到矩陣和從矩陣接收到的數據流，只要你依然對其擁有MARK。你可以現場收聽、觀看或閱讀竊取到的數據，也可以儲存起來以便之後回放/查閱，只要你有用來儲存它們的裝置——你的碟板樂意效勞。

 

窺探的用途USES OF SNOOP

〈窺探〉的用途常常不僅僅止步于偷看偷聽。如果你的目標在打電話（make a commcall），你可以記錄到他聯繫的目標，並通過一個〈矩陣偵察〉動作來在矩陣網路上找到他們。如果他們在100米以內，你立刻在矩陣中發現他們。（如果你所窺探目標的聯繫人正在靜默執行的話，你必須按〈矩陣偵察〉的規則對抗並取得凈成功）

 

偽造指令 SPOOF COMMAND

（複雜動作）

許可權需求：1個MARK（見描述）

檢定：駭入＋直覺【隱匿性】v. 邏輯＋防火墻

你欺騙一臺裝置的所有者認證系統，以使這臺裝置認為你的指令是來自於其所有者的合法指令。你需要對你模仿的圖示（即該裝置的真正所有者——譯註）擁有1個MARK，但並不需要對目標裝置擁有MARK。儘管如此，對抗骰池依然是基於該裝置的屬性。這個動作只對裝置或代理（Agents）生效，對IC、網精、主機、化身或其他任何圖示都無效。

 

切換界面模式 SWITCH INTERFACE MODE 

（簡單動作）

許可權需求：所有者

檢定：無（數據處理類動作）

你把你的視角從AR模式切換到VR模式，或者反過來從VR模式切換到AR模式。切換到VR模式會使你的身體不能動彈，所以別在危險的地方這麼幹。如果你從VR模式切換到AR模式，你失去VR模式下的獎勵主動性骰，見〖改變主動性〗。如果你被鏈鎖（229頁），你不能切換切入模式。

你只能切換自己的接入模式，不能對其他人這麼幹。

 

圖示追蹤 TRACE ICON

（複雜動作）

許可權需求：2個MARK

檢定：計算機＋直覺【數據處理】v. 意志＋隱匿性

你尋找矩陣中目標裝置或化身的物理定位。在對抗檢定中勝出后，只要你依然對目標擁有至少一個MARK，你就可以找到它的位置。〈圖示追蹤〉對主機無效，因為它們一般沒有物理位置；〈圖示追蹤〉同樣無法作用於IC，因為它們只存在於它們的主機之內。（用來順著網線砍人）
